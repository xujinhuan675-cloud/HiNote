# Anchor Gloss 上手说明

## 这是什么

`Anchor Gloss` 是一个基于高亮阅读的 Obsidian 插件。

它的核心体验是：

- 用高亮作为视觉锚点
- 用 AI 为高亮生成贴近原文的解释
- 在原文下方直接显示译文、解析、主干提取或白话改写

当前版本只借鉴 `obsidian-traductor-interlineal` 的显示形态，不包含 PDF 相关能力。

## 安装位置

当前插件目录：

```text
F:\AnchorOS\.obsidian\plugins\anchor-gloss
```

启用方式：

1. 打开 Obsidian
2. 进入 `设置 -> 第三方插件`
3. 关闭安全模式
4. 在已安装插件中启用 `Anchor Gloss`

## 第一次使用前

先在插件设置里配置一个 AI 服务：

1. 打开 `设置 -> Anchor Gloss`
2. 选择你的 AI 提供商
3. 填入对应的 `API Key / URL / Model`
4. 保存设置

如果你本地有 `Ollama`，也可以直接走本地模型。

## 最短使用路径

1. 在笔记里用 `==高亮==` 标记你想理解的句子
2. 打开 `Anchor Gloss` 侧栏视图
3. 找到对应高亮卡片
4. 点击卡片右上角的 AI 按钮
5. 选择一个 Prompt：
   - `贴行翻译`
   - `主干提取`
   - `白话改写`
   - `简明解释`
6. 等待 AI 返回结果
7. 回到原文区域，查看高亮下方自动出现的贴行解释块

## 当前默认 Prompt 说明

- `贴行翻译`
  - 适合英文原句下方直接显示中文译文
- `主干提取`
  - 适合长句拆结构，快速抓主谓宾和逻辑骨架
- `白话改写`
  - 适合拗口句、术语句，转成更好懂的话
- `简明解释`
  - 适合把一句话解释成人能立刻理解的意思

## 显示效果说明

当前版本有两层显示：

- 原文高亮
  - 负责提供明显锚点
- 贴行解释块
  - 直接显示在高亮后方或下方
  - 样式上使用左侧强调线和轻背景，尽量贴近沉浸式翻译的阅读感

## 数据保存方式

AI 结果当前仍然复用评论存储链路，但会带额外标记：

- `source = ai`
- `inline = true`
- `kind = translation / explanation / structure / rewrite`

这样做的好处是：

- 不需要重做整套存储系统
- 能兼容已有评论数据
- 能单独识别哪些内容要“贴行显示”

## 目前已完成

- 独立插件目录已落到 `.obsidian/plugins/anchor-gloss`
- 插件身份已改为 `Anchor Gloss`
- AI 结果已支持按 Prompt 分类成贴行解释
- 编辑模式已支持 block widget 形式的贴行解释块
- 阅读模式已支持高亮后的贴行解释块

## 当前限制

- 还没有做“全文自动挑重点并自动高亮”
- 旧代码里仍保留部分 `HiNote` 命名作为内部结构名，不影响当前使用
- 贴行解释块当前以纯文本方式显示，后续可继续升级成更强的 Markdown 渲染
- 暂不支持 PDF 工作流

## 下一步建议

如果继续迭代，建议按这个顺序做：

1. 增加“全文自动找重点”命令
2. 增加高亮右键菜单或悬浮按钮触发 AI
3. 给贴行解释块加入折叠/展开
4. 区分“译文 / 主干 / 白话 / 解释”的颜色与图标
5. 把贴行解释块升级成 Markdown 渲染

## 构建命令

如果你要继续本地开发：

```bash
npm install
npm run build
```

构建产物会生成到当前插件目录下，Obsidian 重载插件后即可生效。
